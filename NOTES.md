# NOTES

The best-performing model replaced the standard Transformer feed-forward network with a SwiGLU block while keeping the total parameter count below the assignment limit. AdamW was used together with cosine learning-rate decay, a 200-step warmup schedule, weight decay of 0.05, and gradient clipping for stable optimization. Experiments with increasing model capacity resulted in worse generalization under the fixed 2000-step training budget, suggesting that optimization quality was more important than parameter count. A later experiment using Xavier initialization and optimizer parameter grouping also performed worse than the selected configuration. Therefore, the final submission uses the SwiGLU architecture together with the original initialization strategy. AI assistance was used primarily for brainstorming possible experiments, debugging environment issues, and discussing implementation ideas. The final experimental choices, code integration, execution of experiments, interpretation of results, and selection of the submitted model were performed by me based on measured evaluation scores.
